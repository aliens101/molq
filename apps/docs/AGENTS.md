# MolQ documentation instructions

## Product language

- Use "MolQ" for the protocol and "MolQ agent" for the runtime.
- Use "shield sleeve" for Aave allocation and "liquid sleeve" for idle vault assets.
- Distinguish active projected APY from target scenario APY.
- Never describe projected APY as realized or guaranteed yield.
- State that Bybit collateral is offchain and separate from vault custody.

## Writing style

- Use active voice and concise sentences.
- Address the reader as "you" in user workflows.
- Use sentence case for headings.
- Put commands, paths, addresses, and identifiers in code formatting.
- Link contract claims to Mantlescan when practical.

## Content boundaries

- Never include private keys, API secrets, operator credentials, or VPS details.
- Do not imply that MolQ has completed an independent audit.
- Do not claim that the agent can bypass deterministic policy.
- Public GET endpoints may be documented for general use.
- Privileged POST endpoints must be labeled operator-only.
